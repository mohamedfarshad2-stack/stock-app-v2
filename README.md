## About Laravel

Laravel is a web application framework with expressive, elegant syntax. We believe development must be an enjoyable and creative experience to be truly fulfilling. Laravel takes the pain out of development by easing common tasks used in many web projects, such as:

- [Simple, fast routing engine](https://laravel.com/docs/routing).
- [Powerful dependency injection container](https://laravel.com/docs/container).
- Multiple back-ends for [session](https://laravel.com/docs/session) and [cache](https://laravel.com/docs/cache) storage.
- Expressive, intuitive [database ORM](https://laravel.com/docs/eloquent).
- Database agnostic [schema migrations](https://laravel.com/docs/migrations).
- [Robust background job processing](https://laravel.com/docs/queues).
- [Real-time event broadcasting](https://laravel.com/docs/broadcasting).

Laravel is accessible, powerful, and provides tools required for large, robust applications.

## EC2 Auto Deploy

This repo includes:

- `.github/workflows/deploy.yml` to deploy automatically after pushes to `main`
- `deploy.sh` to run the server-side Laravel deploy steps on EC2

One-time setup on EC2:

1. Switch the repo remote from HTTPS to SSH:

```bash
cd /var/www/laravel/app
git remote set-url origin git@github.com:ilhamzacky/stock-app.git
```

2. Generate an SSH deploy key on EC2:

```bash
ssh-keygen -t ed25519 -C "ec2-deploy"
cat ~/.ssh/id_ed25519.pub
```

Add the printed public key to GitHub:

- Repo `Settings`
- `Deploy keys`
- `Add deploy key`
- Enable read access

3. Make sure the deploy script is executable:

```bash
cd /var/www/laravel/app
chmod +x deploy.sh
```

4. Add these GitHub Actions secrets in the repository:

- `EC2_HOST` = your EC2 public IP or domain
- `EC2_USER` = `ubuntu`
- `EC2_SSH_KEY` = the private SSH key used by GitHub Actions to connect to EC2

Suggested deploy key flow:

- Use one SSH key on your local machine to connect to GitHub normally
- Use a separate deploy key on EC2 for `git pull`
- Use a separate private key in GitHub Actions for SSH access into EC2

After setup, every merge or push to `main` will:

1. SSH into EC2
2. Run `deploy.sh`
3. Pull the latest code
4. Install Composer dependencies
5. Run migrations
6. Rebuild Laravel caches
